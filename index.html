¿Qué es la POO?

La Programación Orientada a Objetos (POO) es un paradigma de programación que se basa en el concepto de "objetos", los cuales pueden contener datos y código. Los datos están en forma de campos (conocidos como atributos o propiedades), y el código, en forma de procedimientos (conocidos como métodos).

🔐 Reutilización

El código puede ser reutilizado a través de herencia y composición.

🛡️ Encapsulamiento

Oculta los detalles de implementación y expone solo lo necesario.

🧩 Modularidad

El código se organiza en módulos independientes y manejables.

🔧 Mantenibilidad

Facilita el mantenimiento y la actualización del código.

Clases y Objetos

Una clase es como un plano o plantilla que define las características y comportamientos de algo. Un objeto es una instancia específica de esa clase.

💡 Analogía: Piensa en una clase como el plano de una casa, y el objeto como la casa construida específica basada en ese plano.
// Definición de una clase public class Persona { // Atributos (propiedades) private String nombre; private int edad; private String ciudad; // Constructor completo public Persona(String nombre, int edad, String ciudad) { this.nombre = nombre; this.edad = edad; this.ciudad = ciudad; } // Métodos public void saludar() { System.out.println("Hola, soy " + nombre + " de " + ciudad); } // Getters public String getNombre() { return nombre; } public int getEdad() { return edad; } public String getCiudad() { return ciudad; } } // Creación de objetos usando el constructor completo Persona persona1 = new Persona("Juan", 25, "Madrid"); Persona persona2 = new Persona("María", 30, "Barcelona"); Persona persona3 = new Persona("Carlos", 28, "Valencia"); // Usar los objetos persona1.saludar(); // Hola, soy Juan de Madrid System.out.println(persona2.getNombre() + " tiene " + persona2.getEdad() + " años");

Encapsulamiento

El encapsulamiento es el principio de ocultar los detalles internos de una clase y exponer solo lo necesario a través de métodos públicos. Esto protege los datos y previene modificaciones no autorizadas.

🛡️ Protección

Los datos están protegidos contra accesos no autorizados.

✅ Validación

Permite validar datos antes de modificarlos.

🔌 Flexibilidad

Permite cambiar la implementación sin afectar el código externo.

public class CuentaBancaria { // Atributos privados - encapsulados private double saldo; private String numeroCuenta; private String titular; // Constructor completo public CuentaBancaria(String numeroCuenta, String titular, double saldoInicial) { this.numeroCuenta = numeroCuenta; this.titular = titular; // Validamos que el saldo inicial sea válido if (saldoInicial >= 0) { this.saldo = saldoInicial; } else { this.saldo = 0; } } // Getters public double getSaldo() { return saldo; } public String getNumeroCuenta() { return numeroCuenta; } public String getTitular() { return titular; } // Métodos para operar la cuenta public void depositar(double monto) { if (monto > 0) { saldo += monto; System.out.println("Depósito exitoso: $" + monto); } } public boolean retirar(double monto) { if (monto > 0 && saldo >= monto) { saldo -= monto; System.out.println("Retiro exitoso: $" + monto); return true; } System.out.println("Fondos insuficientes"); return false; } } // Uso del constructor CuentaBancaria cuenta = new CuentaBancaria("123456", "Juan Pérez", 1000.0); System.out.println("Titular: " + cuenta.getTitular()); cuenta.depositar(500.0); cuenta.retirar(200.0);
⚠️ Importante: Usa modificadores de acceso (private, public, protected) para controlar qué puede ser accedido desde fuera de la clase.

Constructor

El constructor es un método especial que se ejecuta automáticamente cuando se crea un objeto. Su propósito es inicializar los atributos del objeto.

Características del Constructor:

  • Tiene el mismo nombre que la clase
  • No tiene tipo de retorno (ni void)
  • Recibe todos los parámetros necesarios para inicializar el objeto
  • Se ejecuta automáticamente al crear un objeto con new
public class Coche { private String marca; private String modelo; private int año; // Constructor completo - recibe todos los parámetros public Coche(String marca, String modelo, int año) { this.marca = marca; this.modelo = modelo; this.año = año; } // Getters public String getMarca() { return marca; } public String getModelo() { return modelo; } public int getAño() { return año; } } // Uso del constructor para crear objetos Coche coche1 = new Coche("Toyota", "Corolla", 2023); Coche coche2 = new Coche("Honda", "Civic", 2024); Coche coche3 = new Coche("Ford", "Mustang", 2025); // Acceder a la información usando getters System.out.println(coche1.getMarca()); // Toyota System.out.println(coche2.getModelo()); // Civic System.out.println(coche3.getAño()); // 2025

Getters y Setters

Los getters y setters son métodos que permiten acceder y modificar los atributos privados de una clase de manera controlada.

Tipo Propósito Convención
Getter Obtener el valor de un atributo getNombreAtributo()
Setter Modificar el valor de un atributo setNombreAtributo(valor)
public class Estudiante { private String nombre; private int edad; private double promedio; // ================== GETTERS ================== public String getNombre() { return nombre; } public int getEdad() { return edad; } public double getPromedio() { return promedio; } // ================== SETTERS ================== // Setter para nombre con validación public void setNombre(String nombre) { if (nombre != null && !nombre.isEmpty()) { this.nombre = nombre; } } // Setter para edad con validación public void setEdad(int edad) { if (edad >= 0 && edad <= 120) { this.edad = edad; } else { throw new IllegalArgumentException("Edad no válida"); } } // Setter para promedio con validación public void setPromedio(double promedio) { if (promedio >= 0.0 && promedio <= 10.0) { this.promedio = promedio; } } } // ================== USO DE GETTERS Y SETTERS ================== Estudiante est = new Estudiante(); // Usando SETTERS para asignar valores est.setNombre("Ana"); est.setEdad(20); est.setPromedio(8.5); // Usando GETTERS para obtener valores System.out.println(est.getNombre()); // Output: Ana System.out.println(est.getEdad()); // Output: 20 System.out.println(est.getPromedio()); // Output: 8.5
💡 Ventaja: Los métodos set permiten validar los datos antes de asignarlos, manteniendo la integridad del objeto. Los métodos get permiten leer los datos sin modificarlos.

toString

El método toString() es un método especial de Java que devuelve una representación en forma de texto de un objeto. Todos los objetos en Java tienen este método porque heredan de la clase Object.

📖 Representación legible

Convierte un objeto a texto para mostrar información importante.

🔍 Debugging

Facilita ver el contenido de un objeto al depurar código.

📤 Logging

Permite registrar objetos en sistemas de logs de forma clara.

¿Qué hace toString() por defecto?

Si no sobrescribes toString(), Java muestra: NombreClase@hexadecimal

// ================== SIN toString() ================== public class Estudiante { private String nombre; private int edad; public Estudiante(String nombre, int edad) { this.nombre = nombre; this.edad = edad; } } // Uso sin toString Estudiante est = new Estudiante("Ana", 20); System.out.println(est); // Output: Estudiante@15db9742 (no es legible) // ================== CON toString() ================== public class Estudiante { private String nombre; private int edad; public Estudiante(String nombre, int edad) { this.nombre = nombre; this.edad = edad; } // Sobrescribir toString() @Override public String toString() { return "Estudiante{nombre='" + nombre + "', edad=" + edad + "}"; } } // Uso con toString Estudiante est = new Estudiante("Ana", 20); System.out.println(est); // Output: Estudiante{nombre='Ana', edad=20}
💡 @Override: Esta anotación le dice a Java que estás sobrescribiendo un método de la clase padre (Object). Es una buena práctica usarla.

Formatos comunes de toString()

No hay una regla estricta, pero estos son los formatos más usados:

// ================== FORMATO 1: Estilo Clases de Java ================== @Override public String toString() { return "Clase{campo1='" + campo1 + "', campo2=" + campo2 + "}"; } // Output: Persona{nombre='Juan', edad=25} // ================== FORMATO 2: Descriptivo ================== @ Override public String toString() { return nombre + " (" + edad + " años)"; } // Output: Juan (25 años) // ================== FORMATO 3: Con salto de línea ================== @Override public String toString() { return "Producto:\n" + " Código: " + codigo + "\n" + " Nombre: " + nombre + "\n" + " Precio: $" + precio; } // Output: // Producto: // Código: 1 // Nombre: Laptop // Precio: $999.99

Ejemplo completo con toString()

public class Libro { private String titulo; private String autor; private int anioPublicacion; private double precio; // Constructor completo public Libro(String titulo, String autor, int anioPublicacion, double precio) { this.titulo = titulo; this.autor = autor; this.anioPublicacion = anioPublicacion; this.precio = precio; } // Getters public String getTitulo() { return titulo; } public String getAutor() { return autor; } public int getAnio() { return anioPublicacion; } public double getPrecio() { return precio; } // Método toString() @Override public String toString() { return "Libro{" + "titulo='" + titulo + "', " + "autor='" + autor + "', " + "anio=" + anioPublicacion + ", " + "precio=$" + precio + "}"; } } // ================== USO DE toString() ================== public static void main(String[] args) { Libro libro = new Libro("El Principito", "Antoine de Saint-Exupéry", 1943, 15.99); // toString() se llama automáticamente con println System.out.println(libro); // Output: Libro{titulo='El Principito', autor='Antoine de Saint-Exupéry', anio=1943, precio=$15.99} // Concatenación con strings String mensaje = "Libro disponible: " + libro; System.out.println(mensaje); // En JOptionPane JOptionPane.showMessageDialog(null, libro.toString()); // Con arreglos Libro[] biblioteca = new Libro[3]; biblioteca[0] = libro; System.out.println(Arrays.toString(biblioteca)); }
⚠️ Importante: toString() debe devolver una representación ÚTIL del objeto, no solo los datos. Incluye la información más importante para identificar el objeto.

toString() con objetos relacionados

Cuando un objeto tiene otros objetos dentro, también llama a sus toString():

public class Direccion { private String calle; private int numero; private String ciudad; public Direccion(String calle, int numero, String ciudad) { this.calle = calle; this.numero = numero; this.ciudad = ciudad; } @Override public String toString() { return calle + " " + numero + ", " + ciudad; } } public class Persona { private String nombre; private Direccion direccion; // Objeto relacionado public Persona(String nombre, Direccion direccion) { this.nombre = nombre; this.direccion = direccion; } @Override public String toString() { // Llama al toString() de Direccion return "Persona{nombre='" + nombre + "', direccion=" + direccion + "}"; } } // Uso Direccion dir = new Direccion("Calle Falsa", 123, "Madrid"); Persona persona = new Persona("Juan", dir); System.out.println(persona); // Output: Persona{nombre='Juan', direccion=Calle Falsa 123, Madrid}

📋 Buenas prácticas para toString()

  • ✅ Incluir información de identificación: Campos que hacen único al objeto
  • ✅ Ser conciso: Solo información importante, no todos los campos
  • ✅ Usar formato claro: Que sea fácil de leer y entender
  • ✅ No llamar a métodos que modifican el estado: toString() no debe modificar el objeto
  • ✅ Manejar null: Si un campo puede ser null, validarlo antes de concatenar
// ================== EJEMPLO CON VALIDACIÓN DE NULL ================== @Override public String toString() { return "Estudiante{" + "nombre='" + nombre + "', " + "edad=" + edad + (carrera != null ? ", carrera='" + carrera + "'" : "") + "}"; } // Si carrera es null: Estudiante{nombre='Ana', edad=20} // Si carrera tiene valor: Estudiante{nombre='Ana', edad=20, carrera='Informática'}
💡 Consejo: En muchos IDEs como Eclipse o IntelliJ, puedes generar automáticamente el método toString() con clic derecho → Generate → toString(). Solo selecciona los campos que quieres incluir.

Enums (Enumeraciones)

Un enum es un tipo de dato especial que permite definir un conjunto de constantes con nombre. Son ideales para representar un conjunto fijo de valores relacionados.

Casos de uso comunes:

  • Días de la semana
  • Estados (activo, inactivo, pendiente)
  • Colores
  • Niveles de usuario
  • Tipos de documentos
// ================== ENUM SIMPLE ================== public enum DiaSemana { LUNES, MARTES, MIERCOLES, JUEVES, VIERNES, SABADO, DOMINGO } // ================== ENUM CON MÉTODOS Y ATRIBUTOS ================== public enum NivelUsuario { ADMINISTRADOR(1, "Tiene acceso total"), MODERADOR(2, "Puede moderar contenido"), USUARIO(3, "Acceso básico"); private final int nivel; private final String descripcion; // Constructor del enum NivelUsuario(int nivel, String descripcion) { this.nivel = nivel; this.descripcion = descripcion; } // Getters public int getNivel() { return nivel; } public String getDescripcion() { return descripcion; } } // ================== USO DE ENUMS ================== // Usar un enum simple DiaSemana hoy = DiaSemana.LUNES; if (hoy == DiaSemana.SABADO || hoy == DiaSemana.DOMINGO) { System.out.println("¡Es fin de semana!"); } // Usar un enum con atributos NivelUsuario nivel = NivelUsuario.ADMINISTRADOR; System.out.println(nivel.getDescripcion()); // Output: Tiene acceso total System.out.println(nivel.getNivel()); // Output: 1 // Recorrer todos los valores de un enum for (DiaSemana dia : DiaSemana.values()) { System.out.println(dia); }

CRUD

CRUD son las operaciones básicas para gestionar datos: Create (Crear), Read (Leer), Update (Actualizar) y Delete (Eliminar).

Operación SQL Descripción
Create INSERT Crear nuevos registros
Read SELECT Leer/consultar registros
Update UPDATE Actualizar registros existentes
Delete DELETE Eliminar registros
// ================== CLASE PRODUCTO ================== public class Producto { private int id; private String nombre; private double precio; // Constructor completo public Producto(int id, String nombre, double precio) { this.id = id; this.nombre = nombre; this.precio = precio; } // Getters public int getId() { return id; } public String getNombre() { return nombre; } public double getPrecio() { return precio; } // Setters public void setNombre(String nombre) { this.nombre = nombre; } public void setPrecio(double precio) { this.precio = precio; } } // ================== GESTOR DE PRODUCTOS ================== public class GestorProductos { // Arreglo de productos (máximo 50) private Producto[] productos; private int cantidad; // Constructor public GestorProductos() { productos = new Producto[50]; cantidad = 0; } // ================== CREATE ================== public boolean crearProducto(int id, String nombre, double precio) { if (cantidad >= 50) { System.out.println("No hay espacio para más productos"); return false; } productos[cantidad] = new Producto(id, nombre, precio); cantidad++; System.out.println("Producto creado: " + nombre); return true; } // ================== READ ================== // Listar todos los productos public void listarProductos() { if (cantidad == 0) { System.out.println("No hay productos registrados"); return; } System.out.println("=== LISTA DE PRODUCTOS ==="); for (int i = 0; i < cantidad; i++) { System.out.println(productos[i].getId() + ". " + productos[i].getNombre() + " - $" + productos[i].getPrecio()); } } // Obtener producto por ID public Producto obtenerPorId(int id) { for (int i = 0; i < cantidad; i++) { if (productos[i].getId() == id) { return productos[i]; } } return null; } // ================== UPDATE ================== public boolean actualizarProducto(int id, String nuevoNombre, double nuevoPrecio) { for (int i = 0; i < cantidad; i++) { if (productos[i].getId() == id) { productos[i].setNombre(nuevoNombre); productos[i].setPrecio(nuevoPrecio); System.out.println("Producto actualizado"); return true; } } System.out.println("Producto no encontrado"); return false; } // ================== DELETE ================== public boolean eliminarProducto(int id) { // Buscar el producto int posicion = -1; for (int i = 0; i < cantidad; i++) { if (productos[i].getId() == id) { posicion = i; break; } } if (posicion == -1) { System.out.println("Producto no encontrado"); return false; } // Mover los productos una posición atrás for (int i = posicion; i < cantidad - 1; i++) { productos[i] = productos[i + 1]; } // Eliminar la referencia y reducir cantidad productos[cantidad - 1] = null; cantidad--; System.out.println("Producto eliminado"); return true; } } // ================== USO DE CRUD ================== GestorProductos gestor = new GestorProductos(); // CREATE - Crear productos gestor.crearProducto(1, "Laptop", 999.99); gestor.crearProducto(2, "Mouse", 25.50); // READ - Listar productos gestor.listarProductos(); // READ - Obtener un producto específico Producto p = gestor.obtenerPorId(1); if (p != null) { System.out.println("Producto encontrado: " + p.getNombre()); } // UPDATE - Actualizar un producto gestor.actualizarProducto(1, "Laptop Pro", 1199.99); // DELETE - Eliminar un producto gestor.eliminarProducto(2); // Ver resultado final gestor.listarProductos();
💡 Buenas prácticas: Siempre valida los datos antes de crear o actualizar, y verifica si el elemento existe antes de actualizar o eliminar.

Elementos de un Diagrama de Clase:

  • Clase: Rectángulo dividido en 3 secciones (nombre, atributos, métodos)
  • Atributos: Variables de instancia con visibilidad (+, -, #)
  • Métodos: Funciones de la clase
  • Relaciones: Asociación, herencia, implementación, etc.

🔗 Relaciones entre Clases

En POO, las clases pueden relacionarse entre sí de diferentes maneras. Estas relaciones determinan cómo las clases interactúan y se comunican entre sí.

1. Asociación

Qué es: La relación más básica. Una clase "conoce" a otra y puede usarla.

// ================== ASOCIACIÓN CON ROL Y MULTIPLICIDAD ================== // Rol: inscrito (el estudiante está inscrito en cursos) // Multiplicidad: 1 estudiante se puede inscribir en muchos cursos (*) public class Estudiante { private String nombre; private Curso[] cursos; // Asociación: * cursos (muchos) private int cantidadCursos; public Estudiante(String nombre) { this.nombre = nombre; cursos = new Curso[5]; cantidadCursos = 0; } // El estudiante se inscribe en un curso public void inscribirse(Curso curso) { if (cantidadCursos < 5) { cursos[cantidadCursos++] = curso; System.out.println(nombre + " se inscribió en " + curso.getNombre()); } } public void mostrarCursos() { System.out.println(nombre + " está inscrito en:"); for (int i = 0; i < cantidadCursos; i++) { System.out.println(" - " + cursos[i].getNombre()); } } } // Rol: imparte (el curso es impartido por profesores) // Multiplicidad: 1 curso tiene muchos estudiantes (*) public class Curso { private String nombre; public Curso(String nombre) { this.nombre = nombre; } public String getNombre() { return nombre; } } // Uso de la asociación Estudiante est = new Estudiante("Ana"); Curso c1 = new Curso("Programación"); Curso c2 = new Curso("Base de Datos"); est.inscribirse(c1); est.inscribirse(c2); est.mostrarCursos();

2. Composición (Containment)

Qué es: Una clase está "compuesta" por otras clases. Relación fuerte "tiene un".

En diagrama: Línea con rombo lleno ——◆

⚠️ IMPORTANTE: El rombo va en el lado de la clase que CONTIENE (el "padre" o todo).

En código: La clase contenedora tiene objetos de otra clase como atributos.

Característica: Si el objeto padre se destruye, los hijos también.

Auto
◆————
rombo en el contenedor
Motor
// ================== COMPOSICIÓN (CONTAINMENT) ================== // Clase que será parte de otra (el "todo" no existe sin esta parte) public class Motor { private int caballosFuerza; private String tipoCombustible; public Motor(int caballosFuerza, String tipoCombustible) { this.caballosFuerza = caballosFuerza; this.tipoCombustible = tipoCombustible; } public void encender() { System.out.println("Motor encendido: " + caballosFuerza + " HP"); } } // Clase CONTENEDORA (tiene el rombo en el diagrama) public class Auto { private String marca; private Motor motor; // COMPOSICIÓN: Auto TIENE un Motor // Constructor completo public Auto(String marca, Motor motor) { this.marca = marca; this.motor = motor; // El motor es parte del auto } public void encender() { System.out.println("Encendiendo auto " + marca); motor.encender(); // Usa su motor } } // Uso de la composición // El motor se crea específicamente para este auto Motor motorV8 = new Motor(400, "Gasolina"); Auto mustang = new Auto("Ford Mustang", motorV8); mustang.encender(); // Output: // Encendiendo auto Ford Mustang // Motor encendido: 400 HP // Si destruyes el auto, el motor también deja de existir // No puedes sacar el motor de este auto y ponerlo en otro
💡 Composición (Containment):
• El rombo ◆ va en el lado del CONTENEDOR (Auto)
• El motor NO puede existir sin el auto
• Relación de dependencia fuerte (todo-parte)
• Si destruyes el auto, el motor también se destruye

3. Agregación (Sharing)

Qué es: Similar a la composición pero más débil. También es "tiene un" pero los objetos pueden existir independientemente.

En diagrama: Línea con rombo vacío ——◇

⚠️ IMPORTANTE: El rombo va en el lado de la clase que AGREGA (el "todo" o grupo).

En código: La clase agrupadora tiene objetos de otra, pero estos pueden existir por separado.

Equipo
◇————
rombo en el agregador
Jugador
// ================== AGREGACIÓN (SHARING) ================== // Clase que puede existir independientemente public class Jugador { private String nombre; private int numeroCamiseta; public Jugador(String nombre, int numeroCamiseta) { this.nombre = nombre; this.numeroCamiseta = numeroCamiseta; } // Getters public String getNombre() { return nombre; } public int getNumero() { return numeroCamiseta; } } // Clase AGREGADORA (tiene el rombo en el diagrama) public class Equipo { private String nombre; private Jugador[] jugadores; // AGREGACIÓN: Equipo TIENE Jugadores private int cantidad; // Constructor public Equipo(String nombre) { this.nombre = nombre; jugadores = new Jugador[11]; cantidad = 0; } // Agregar jugador al equipo (compartir) public void agregarJugador(Jugador jugador) { if (cantidad < 11) { jugadores[cantidad++] = jugador; System.out.println(jugador.getNombre() + " agregado a " + nombre); } else { System.out.println("Equipo completo"); } } // Mostrar jugadores del equipo public void mostrarJugadores() { System.out.println("Jugadores de " + nombre + ":"); for (int i = 0; i < cantidad; i++) { System.out.println("- " + jugadores[i].getNumero() + ". " + jugadores[i].getNombre()); } } } // Uso de la agregación (sharing) // Los jugadores pueden existir sin el equipo Jugador j1 = new Jugador("Messi", 10); Jugador j2 = new Jugador("Di María", 11); // Un jugador puede pertenecer a un equipo Equipo equipo1 = new Equipo("Argentina"); equipo1.agregarJugador(j1); equipo1.agregarJugador(j2); equipo1.mostrarJugadores(); // El jugador puede cambiarse a otro equipo (sharing) Equipo equipo2 = new Equipo("PSG"); equipo2.agregarJugador(j1); // Messi puede jugar en otro equipo // Si se disuelve el equipo, los jugadores siguen existiendo // y pueden unirse a otro equipo
💡 Agregación (Sharing):
• El rombo ◇ va en el lado del AGREGADOR (Equipo)
• Los jugadores PUEDEN existir sin el equipo
• Relación débil (agrupación, no todo-parte)
• Un jugador puede pertenecer a varios equipos en diferentes momentos

Tipo Símbolo UML Concepto ¿Dónde va el rombo? Ejemplo
Asociación —— "Conoce a" Sin rombo Estudiante inscribe Curso
Composición ◆—— "Tiene un" (fuerte) En el CONTENEDOR Auto tiene Motor
Agregación ◇—— "Tiene un" (débil) En el AGREGADOR Equipo tiene Jugadores
💡 Regla mnemotécnica:
• El rombo SIEMPRE va del lado que "TIENE" o "AGRUPA"
• ◆ Composición: rombo LLENO = relación FUERTE (containment)
• ◇ Agregación: rombo VACÍO = relación DÉBIL (sharing)

// ================== SISTEMA DE TIENDA ================== // Clase Producto (para composición y agregación) public class Producto { private int codigo; private String nombre; private double precio; public Producto(int codigo, String nombre, double precio) { this.codigo = codigo; this.nombre = nombre; this.precio = precio; } // Getters public int getCodigo() { return codigo; } public String getNombre() { return nombre; } public double getPrecio() { return precio; } } // ================== COMPOSICIÓN (Containment) ================== // Compra CONTIENE productos (rombo en Compra) public class Compra { private String fecha; private Producto[] productos; // Compra TIENE productos (*) private int cantidad; public Compra(String fecha) { this.fecha = fecha; productos = new Producto[10]; cantidad = 0; } // Agregar producto a la compra public void agregarProducto(Producto prod) { if (cantidad < 10) { productos[cantidad++] = prod; } } public double getTotal() { double total = 0; for (int i = 0; i < cantidad; i++) { total += productos[i].getPrecio(); } return total; } } // ================== ASOCIACIÓN ================== // Cliente REALIZA compras (sin rombo) public class Cliente { private String nombre; private String dni; public Cliente(String nombre, String dni) { this.nombre = nombre; this.dni = dni; } // ASOCIACIÓN: el cliente realiza una compra temporalmente public void realizarCompra(Compra compra) { System.out.println("Cliente: " + nombre); System.out.println("Total de compra: $" + compra.getTotal()); } // Getters public String getNombre() { return nombre; } } // ================== AGREGACIÓN (Sharing) ================== // Tienda AGRUPA productos en inventario (rombo en Tienda) public class Tienda { private String nombre; private Producto[] inventario; // Tienda AGRUPA productos (*) private int cantidad; public Tienda(String nombre) { this.nombre = nombre; inventario = new Producto[100]; cantidad = 0; } // Agregar producto al inventario public void agregarProducto(Producto prod) { if (cantidad < 100) { inventario[cantidad++] = prod; System.out.println("Producto agregado: " + prod.getNombre()); } } // Mostrar inventario public void mostrarInventario() { System.out.println("=== INVENTARIO DE " + nombre + " ==="); for (int i = 0; i < cantidad; i++) { System.out.println(inventario[i].getCodigo() + ". " + inventario[i].getNombre() + " - $" + inventario[i].getPrecio()); } } } // ================== USO DEL SISTEMA ================== public static void main(String[] args) { // Crear productos (existen independientemente) Producto p1 = new Producto(1, "Laptop", 999.99); Producto p2 = new Producto(2, "Mouse", 25.50); // AGREGACIÓN: Tienda agrupa productos (sharing) Tienda tienda = new Tienda("TechStore"); tienda.agregarProducto(p1); tienda.agregarProducto(p2); // COMPOSICIÓN: Crear una compra que contiene productos Compra compra = new Compra("10/03/2026"); compra.agregarProducto(p1); // Mismo producto se puede compartir compra.agregarProducto(p2); // ASOCIACIÓN: Cliente realiza una compra Cliente cliente = new Cliente("Juan Pérez", "12345678"); cliente.realizarCompra(compra); // Mostrar inventario de la tienda tienda.mostrarInventario(); // Diferencia clave: // - Si se anula la compra, los productos SIGUEN existiendo (están en la tienda) // - Si cierra la tienda, los productos SIGUEN existiendo (pueden venderse en otra tienda) // - La compra es solo una asociación temporal con el cliente }
💡 Resumen del ejemplo con roles y multiplicidades:
• Asociación: Cliente(1) —realiza——> Compra(*) - Sin rombo
• Composición: Compra(1) ◆—contiene——> Producto(*) - Rombo en Compra (containment)
• Agregación: Tienda(1) ◇—vende——> Producto(*) - Rombo en Tienda (sharing)

Ejercicio 1: Clase Estudiante

Fácil
Objetivo: Crea una clase Estudiante con los siguientes requisitos:

  • Atributos privados: nombre (String), nota (double)
  • Constructor que reciba nombre y nota
  • Getter para cada atributo
  • Setter para la nota (que solo acepte valores entre 0 y 10)
  • Método estaAprobado() que retorne true si la nota >= 6
💡 Pista:
• Los atributos privados se declaran con private
• El constructor tiene el mismo nombre que la clase
• El setter debe validar: if (nota >= 0 && nota <= 10)
• El método estaAprobado retorna: return this.nota >= 6;

Ejercicio 2: Gestor de Productos con CRUD

Medio
Objetivo: Completa los métodos CRUD en la clase GestorProductos:
  • Usa un String[] para almacenar nombres de productos (máximo 10)
  • CREATE: Método agregarProducto(String nombre) - agrega si hay espacio
  • READ: Método listarProductos() - muestra todos los productos
  • UPDATE: Método actualizarProducto(int indice, String nuevoNombre)
  • DELETE: Método eliminarProducto(int indice) - deja null en esa posición
💡 Pista:
• Para agregar: verifica if (cantidad < 10) y asigna productos[cantidad++] = nombre;
• Para listar: usa un for hasta cantidad y acumula en un String
• Para actualizar: verifica if (indice >= 0 && indice < cantidad)
• Para eliminar: asigna null y mueve los elementos restantes
